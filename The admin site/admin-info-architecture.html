Core Workflows

These are the workflows the prototype should simulate before backend work begins.

KYC Approval Flow

Client submits documents, admin reviews requirements, approves or rejects, then the client dashboard unlocks funding and withdrawal capabilities.

Deposit Confirmation Flow

Client creates a deposit request, admin verifies payment reference, marks it confirmed, and wallet balance updates on the client side.

Withdrawal Approval Flow

Client requests withdrawal, admin checks KYC, balance, risk, and destination, then approves, holds, rejects, or escalates.

Portfolio Publishing Flow

Admin creates or edits a portfolio product, reviews risk and copy, publishes it, and it becomes available on the investor dashboard and public site if enabled.

Payout Posting Flow

Admin records dividend or profit payout, selects wallet credit or reinvestment, confirms posting, and the client dashboard reflects the entry.

Options Access Flow

Client applies for advanced access, admin reviews suitability and risk profile, assigns access level, and restricted options become available only if approved.

Admin Roles

The UI should be designed with permissions in mind, even while the first pass remains static.

Super Admin

Full platform control, role management, settings, audit review, and final override authority.

Compliance Officer

KYC review, risk exceptions, options suitability, account restrictions, and compliance notes.

Finance Operations

Deposits, withdrawals, fees, reconciliation, payout posting, and settlement tracking.

Portfolio Manager

Portfolio products, allocations, investment mandates, performance notes, and published portfolio content.

Support Agent

Support tickets, user communication, client issue tracking, and operational handoffs.

Read-Only Auditor

Audit logs, reports, client records, admin decisions, and export history without mutation access.
